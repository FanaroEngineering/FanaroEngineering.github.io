VS Code has taken the programming world by storm. Corey Schafer, a much more experienced programmer than I mentions in one of his videos that, in a coding event he goes to regularly, where he used to see a myriad of editors, VS Code is now almost unanimous.

I believe there are 2 major factors for its success:

  1. Lightweight: whereas previous good editors would take a lot processing and RAM, which would make some people even stick to very old school or bare bones editors like Emacs.
  2. Beautiful and with no clutter: its minimalist design focuses on having no needless features and the overall design feels sharp and modern.
  3. OS Independent: VS Code was programmed using ElectronJS, a JavaScript layer on top of the programming languages used for developing software for different operating systems. This allows for almost no OS incompatibility and grants the software a cleaner look.
  4. More importantly: it's open source (you can simply go to its Github repository and see how they coded it). Open Source code makes the code available for users to customize and point out bugs. In practice, in general, software companies have experienced great boosts in development and user satisfaction by open sourcing their codes. This is a lesson taken from the previous and fairly recent editor that planted the seeds for VS Code: Atom.

VS Code adopts JSON for its settings, something seen nowadays as a best practice, as it is very simple, lightweight and, more importantly, conventional — which makes people's management much easier.

However, the standard configuration is to open an UI to deal with configuration. This can thankfully be changed by placing "workbench.settings.editor" : "json" in the user's configuration, which you can access by typing Ctrl + Shift + P, looking for settings and then choosing Open User Settings (JSON). The default settings are only so you can take a look at what's available; if you see something you want to change, override it when placing it in the user settings. For example, you can use my configurations by simply copying my settings.json file content into your user's configuration. If this looks all too complicated, there are several tutorials on Youtube (including one from Corey Schafer that's very nice) and there's also the official documentation, which is not as unfriendly as your regular software docs.

I've added comments to all of my customization, but the ones I feel aggregate the most value are:
  • editor.acceptSuggestionOnEnter: by setting this to "off", you avoid pressing enter and accidentally inserting one of the suggestions. Now suggestions will only happen when pressing Tab.
  • window.titleBarStyle: this ditches your OS title bar when set to "custom"; overall a cleaner visual.
  • workbench.editor.enablePreview : by default, VS Code opens previews of the files; only when you edit them do they become actual editor windows. If you wish to open multiple files then, VS Code will keep overriding the same tab, which is not typically what you want.
  • editor.minimap.renderCharacters : the characters in the Mini Map are... mini. So what's the point in rendering them? You just want an outline, the sky's view, and not rendering the characters does exactly that.
  • editor.fontLigature : only some fonts enable this feature, and one of them is the Fira Code font. Font Ligatures create more visually pleasing and concise symbols for coding syntax — for example, the usual != different symbol becomes the actual mathematical different symbol: â‰ . Under the hood, these symbols stay the same, it's just a visual feature.
  • explorer.autoReveal : VS Code autofocuses the file explorer to the current file you're editing. This can be very annoying at (most) times, so you can disable it with this configuration.
  • workbench.settings.editor : by choosing "json" here, you force VS Code to open the JSON version of the configurations whenever you press Ctrl + , or open the settings through the command palette.
  • explorer.confirmDragAndDrop : you don't have to confirm every file manipulation you make in the file explorer tab.
  • extensions.ignoreRecommendations : ignore all of the software recommendations VS Code gives you. They can be useful at times, but they are annoying more often than not.
  • editor.multiCursorModifier : this cool little feature lets you Ctrl + Click on multiple lines and then make changes to them at the same time. Very practical sometimes, and, more importantly, awesome.
  • Intellicode
    • This extension uses the power of AI to learn how people code on Github and then give you back suggestions of how to fill yours.
  • Code Runner
    • Run various languages' files through one engine and usually one shortcut. Very cool if you're using a not so customized environment.
    • Doesn't have that great support for changing environment variables.
  • One Dark Pro
    • In my humble opinion, the best theme ever for coding. It's dark, and doesn't have that big a contrast, which makes it easy on the eyes.
    • It became incredibly popular with the rise of the Atom Editor (which was debunked by VS Code).
  • multi-command
    • As is pointed out in this StackOverflow post, don't use the Macros extension, it is quite outdated.
    • This amazing extension lets you concatenate shortcut commands, a desperately needed feature to the keybindings.json file.
    • I have one example on my settings that uses alt+t to copy-paste code and execute it on the Integrated Terminal, while also focusing on it.
  • Docker
    • For those of you who need to create virtual environments and packages that work independently of the user's configurations, Docker is a must. And this extension offers much more than simply a linter, while also being maintained by Microsoft itself.
  • Material Icon Theme
    • A cool alternative is vscode-icons, it goes very well with VS Code's standard dark theme.
  • SQL Server
    • An extension created by Microsoft themselves to manage MSSQL databases directly through the editor.
    • There is also a specific editor that apparently uses VS Code's design that was also created by Microsoft: Azure Data Studio.
  • SQLTools
    • This very cool extension lets you manage all of them database types through in one go. Simply amazing.
  • Prettify JSON
    • Easily format JSON files.
  • Python
    • Microsoft's Official Python Extension.
    • Special support for Python debugging, linting, etc. There is also support for Jupyter Notebooks inside VS Code itself.
  • Rainbow CSV
    • Incredibly useful if you have to work with CSVs with tons of columns. It also gives you the name of the column if you point your mouse on top of a specific item.
  • reStructuredText
    • reStructuredText is similar to Markdown. But it's very common in Python documentation extensions, such as Sphinx. So it's very useful to have syntax and linting support for it — though, admittedly, this extension could be better.

Customizing shortcuts is something I haven't put that much effort in. Besides the fact that VS Code has already many useful ones, it's very difficult to come up with a good key combination without overriding an existing (and probably useful) one. Nonetheless, I have been able to add 2 that are very useful — I really don't understand why they don't come out of the box —: switching through terminals (there is already one for focusing on the terminal tab itself though: Ctrl + `).

And I've also managed to add shortcuts for: copy-pasting code to the integrated terminal and executing it; copy-pasting code to the terminal and executing it while also focusing on the integrated terminal. For the last one, it was necessary to use a macros extension, since there was a need for the concatenation of shortcuts. In order to create it: add this to your settings.json file: 
        multiCommand.commands: [
    {
        "command" : "multiCommand.copyPasteTerminalAndFocus",
        "sequence": [
            {
                "command" : "workbench.action.terminal.sendSequence"
                "args" : {"text" : "${selectedText}\u000D"} 
                // `\u000D` is a `Return`
            },
            "workbench.action.terminal.focus"
        ]
    }
]

And then create the shortcut in the keybindings.json file by adding: 

        {   // Copy Paste to the Integrated Terminal and Focus on it 
    "key" : "alt+t",
    "command" : "multiCommand.copyPasteTerminalAndFocus"
}
