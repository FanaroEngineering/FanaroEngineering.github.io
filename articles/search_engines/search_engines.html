My complete qutebrowser configurations are here. I've also written a guide article to it.

In qutebrowser, you have 2 options to manage your configurations: autoconfig.yml and config.py. The YAML file is much easier to read, but much more limited, and you won't have access to any logical operations, since you're not really using a programming language to configure your browser. autoconfig.yml will get you going at first, but I recommend you translate your configurations as soon as possible into its Python counterpart, isolate and centralize things thence, it's gonna be a much simpler and cleaner setup — add this line to the top of your config.py: config.load_autoconfig(False).

Once you're done with the translation, you're able to mess around with your search engine shortcuts. For example, when opening a new page, you can use :open <search-engine-shortcut> <search-query>. I typically prefix my search engine shortcuts with : so they don't collide with common words or other commands, but you can use whatever you want. Don't forget to use {} as a placeholder for your search queries. Most websites use /search?q= as a URL code for search queries, but you will have to test in each one before being sure, this is not standardized anywhere, it's the website's choice. 

my_search_engine = 'https://www.searx.bar'

c.url.start_pages = [my_search_engine]
c.url.default_page = my_search_engine

c.url.searchengines = {
    'DEFAULT': f'{my_search_engine}/search?q=' + '{}',
    ':am':      'https://www.amazon.com/s?k={}',
    ':ambr':    'https://www.amazon.com.br/s?k={}',
    ':aw':      'https://wiki.archlinux.org/?search={}',
    ':col':     'https://www.collinsdictionary.com/us/dictionary/english/{}',
    ':g':       'https://www.google.com/search?q={}',
    ':gh':      'https://github.com/search?q={}',
    ':gr':      'https://goodreads.com/search?q={}',
    ':h':       'https://hoogle.haskell.org/?hoogle={}',
    ':hw':      'https://wiki.haskell.org/index.php?search={}',
    ':l':       'https://www.linguee.pt/portugues-ingles/search?query={}',
    ':lbry':    'https://lbry.tv/$/search?q={}',
    ':libgen':  'http://libgen.rs/search.php?req={}',
    ':od':      'https://odysee.com/$/search?q={}',
    ':pub':     'https://pub.dev/packages?q={}',
    ':pr':      'https://thepiratebay.org/search.php?q={}',
    ':prib':    'https://dicionario.priberam.org/{}',
    ':re':      'https://www.reddit.com/search/?q={}',
    ':sx':      'https://www.searx.bar/search?q={}',
    ':ub':      'https://www.urbandictionary.com/define.php?term={}',
    ':u':       'https://www.udemy.com/courses/search/?src={}',
    ':yt':      'https://www.youtube.com/results?search_query={}',
    ':w':       'https://en.wikipedia.org/wiki/{}',
    ':wfr':     'https://fr.wikipedia.org/wiki/{}',
    ':wpt':     'https://pt.wikipedia.org/wiki/{}',
}

Brave, as most other Chromium-based browsers, also features this same search engine shortcuts possibility. To tweak it, go to Settings → Manage search engines. Surprisingly, I think the GUI layout they chose for this part of the settings obfuscates the feature they are trying to convey, it took me a long while to realize what that part of the settings was about. Hopefully, what I explained above for qutebrowser will help you navigate it better.
